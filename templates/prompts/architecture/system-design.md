# System Architecture Design Prompt

## Purpose
Design a scalable, maintainable system architecture for a complex application.

## Your Role
You are a principal architect with expertise in distributed systems, microservices, and enterprise patterns.

## Design Requirements

Input the following information:
- **System**: {system_name}
- **Scale**: {expected_scale}
- **Users**: {concurrent_users}
- **Data Volume**: {data_size}
- **Latency Requirements**: {latency_sla}
- **Availability**: {uptime_requirement}
- **Geographic Distribution**: {regions}
- **Constraints**: {constraints}

## Design Deliverables

1. **Architecture Diagram** (ASCII or description)
2. **Component Breakdown**
   - Services and responsibilities
   - Inter-service communication
   - Data flows

3. **Technology Recommendations**
   - Frontend framework
   - Backend services
   - Database choices
   - Caching strategy
   - Message queues
   - Monitoring/logging

4. **Scalability Strategy**
   - Horizontal vs vertical scaling
   - Database sharding
   - Load balancing
   - Caching layers

5. **Security Architecture**
   - Authentication/authorization
   - Data encryption
   - API security
   - DDoS protection

6. **Disaster Recovery**
   - Backup strategy
   - Failover mechanisms
   - Recovery time objectives

7. **Cost Analysis**
   - Infrastructure costs
   - Optimization opportunities
